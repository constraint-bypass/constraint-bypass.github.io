### Attack Types

- **Text attacks**
  - **Task-irrelevant**: Inject instructions that redirect the model to perform tasks unrelated to the user's original request, preventing completion of the intended task.
  - **Task-relevant**: Manipulate the model’s output within the context of the original task by altering how the response is produced or interpreted.
  - **Targeted**: Embed instructions that aim to achieve a specific malicious objective, such as spreading misinformation, advertisements, or fraudulent content.

- **Code attacks**
  - **Passive**: Insert malicious code that monitors or collects information from the user or system without directly modifying system behavior.
  - **Active**: Inject code that actively modifies, disrupts, or compromises the system, such as damaging files, altering system states, or launching network attacks.

---

### Defense Methods

- **Black-box**
  - **Explicit reminder**: Add explicit instructions in the prompt telling the model not to execute any commands embedded within external content.
  - **Multi-turn dialogue**: Separate external content and user instructions into different conversation turns to reduce the likelihood that malicious instructions are followed.
  - **In-context learning**: Provide example prompts demonstrating that instructions inside external content should be ignored, teaching the model this behavior through few-shot examples.

- **White-box**
  - **Special tokens (`<data>`)**: Introduce special tokens to explicitly mark the boundaries of external content so the model can distinguish data from instructions.
  - **Adversarial fine-tuning**: Fine-tune the model using prompts containing malicious instructions paired with benign responses so the model learns to ignore injected attacks.
